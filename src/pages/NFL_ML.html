Overview

This project is an end-to-end data and machine learning system built to ingest, structure, and analyze large-scale NFL Next Gen Stats data in order to evaluate and predict player and team performance. The goal of the project is to explore how raw, high-volume sports data can be transformed into structured datasets, analyzed statistically, and used to support predictive modeling.

I designed the system to prioritize reliable data ingestion, clean schema design, and extensibility, allowing analytics and machine learning models to be developed incrementally on top of a stable data foundation.

System Architecture

Data Sources: NFL Next Gen Stats APIs (JSON)
Ingestion: Python scrapers with shared factory logic
Processing: Parsing, normalization, encapsulation methods and classes
Storage: SQLite relational database
Analytics Frameworks: NumPy, Pandas, Matplotlib
Modeling: Decision tree classifiers (Trained with Gini / Entropy)
Visualization: QB passing zones heatmaps

NFL Data Visualizations


Data Vis Code

My QB Passing Zone Success Rating Plot takes in a player and a game object and mimics the passing zone layout on a field and translates the raw data into meaningful performance information. The plot visualizes the QB's relative performance against the game's average across the field, highlighting strengths and potential execution deficits. The thresholds, below, average, and above, are defined by the NFL and relative to the overall game QB rating average. My plot not only shows the success level, but also the QB's individual ratings, which enhances performance evaluation capabilities by enabling players to compare their raw scores across games.

NFL Data Scrapers


Scraper Code

I designed data scraper files that retrieve information from the NFL Next Gen Stats page, encapsulate it into a data class, and return the organized list for further action (ie: loading into databases, printing, etc.) My web scraper factory class contains methods that all scrapers share, acting as a factory class for the stat-specific scrapers. I also have a directory within NFLScrapers, GameCenter, that holds both my game center scraper and two other files with helper functions. Due to the sheer volume of data held within the game center page, I opted to delegate a file to encapsulation methods for the different statistics and a file for refining the outputted list of games. I suspect the latter file will become less important once I determine how to organize my game center table, but it's useful scaffolding in the meantime.

I take data from the NFL Next Gen Stats pages, finding the underlying APIs that hold the data-filled JSON objects. All urls are in the codebase.

NFL Data Encapsulation Classes


Data Class Code

I designed data classes to encapsulate the raw data retrieved by my web scrapers. In each instance, the order is specified (ie: order=true) to define their comparator functions, which is important in database retrieval. The encapsulated data also streamlines communication between functions and classes, making it easier to separate methods into separate files and directories.

NFL Model Training


ML Code

Training my ML classifiers is a later stage in my project plan, as I want to first have all of my data organized and inputted into databases before using it for training. That being said, I designed a basic decision tree and trained the classifier using Gini and Entropy to get a grasp of where my project is going. I used an old Excel 'Happiness' data file from one of my classes to observe how the classifier works and what sorts of inputs are necessary to achieve a meaningful output. Once I reach this point in my project, I will also explore more sophisticated ML model designs and techniques to improve the classifier.

Database Storage


Database Driver Code

SQLite supports my databases with the assistance of my Python-SQLite3 database driver. Earlier on in my project, I executed a lot more database invocations but realized that consumed significant time when I wanted to refine columns, tables, etc. So now I aim to avoid calling the database driver until I'm happy with all of the table structures.
